## 📊 核心功能

### **1. 多模型并行调用**

**同时调用：**
- **Neo**（本地，4D 压缩 + 进化视角）✅
- **Gemini**（CLI/API，Google 视角）⏳ Phase 2
- **Grok**（CLI/API，工程师视角）⏳ Phase 2

**Phase 1 状态：**
- ✅ Neo：模拟响应（Phase 2 接入 sessions_send）
- ⚠️ Gemini：需要安装 `gemini` CLI（`brew install steipete/tap/summarize`）
- ⚠️ Grok：需要 `XAI_API_KEY` 环境变量

**Phase 1 降级保护：**
- 如果 CLI 不可用，使用模拟响应
- 保证系统可测试，不阻塞开发

---

### **2. 4D 压缩汇总**

**自动将三方回复压缩为 4D 向量：**

| 维度 | 内容 |
|------|------|
| **选择力** | 三方共识的方向/建议 |
| **影响力** | 可立即执行的行动 |
| **反馈力** | 需要注意的风险/警告 |
| **体验力** | 核心洞察/智慧 |

**Phase 1：** 简单 4D 结构  
**Phase 2：** 接入 `uptef-4d-compression-smart-router`

---

### **3. 宝石袋集成（Phase 3）**

**自动扫描 GEM_BAG.md：**

```bash
# 查看待完成任务
warrior gems --pending

# 标记完成
warrior gem complete [宝石 ID]
```
